fix: Resolve weather data parsing from Python API output

Issue: Weather data was not being parsed correctly from Python subprocess.
The system() call was executing Python before popen(), and the command
variable was uninitialized when passed to popen(), resulting in no data
in the pipe.

- Removed redundant system() call that executed Python incorrectly
- Properly initialized command string with snprintf() before popen()
- Added direct sscanf parsing in main.c instead of relying on separate function
